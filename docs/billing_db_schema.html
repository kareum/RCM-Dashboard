구현 순서

1
기반 설정
users → clinics → api_sync_log → sync_settings
2
외부 데이터 수신 + 마스터 (Full sync)
patients_raw → patients → patient_clinic_assignments → billing_raw → cpt_rates
신규=INSERT · 변경=UPDATE+hash갱신 · 삭제감지=is_active false · 클리닉이동=assignments 이력
3
Billing 핵심 흐름
claims → claim_cpt_lines → eob_files → era_matches → invoices → invoice_claims → payments
4
이력 + 알림
claim_status_history → claimmd_submissions → invoice_history → revenue_split_history → notifications
동기화 전략 요약
대상 방식 자동 주기 수동 트리거 비고
clinics Full sync 매일 03:00 AM 설정 화면 Sync now 변경 빈도 낮음
patients Full sync 매일 03:00 AM 설정 화면 Sync now 클리닉 이동 시 assignments 이력 생성
billing Full sync 매일 03:00 AM 설정 화면 Sync now billing_month 기준 비교
처리 규칙: 신규 external_id → INSERT  ·  필드값 변경(hash 불일치) → UPDATE  ·  원서버에서 사라짐 → is_active = false (삭제 X)
